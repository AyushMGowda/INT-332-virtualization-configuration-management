# Docker Volumes:

Docker Volumes are a storage mechanism used to persist data generated by containers. Normally, data inside a container is temporary and may be lost when the container is removed. Volumes solve this problem by storing data outside the container’s filesystem, allowing it to remain available even after containers stop or are deleted.
Volumes are managed by Docker and provide a reliable way to share data between containers or store application data permanently. They are commonly used for databases, logs, and application files that need to survive container restarts.
Using volumes also improves data management, backup, and portability, since the stored data is separated from the container itself.

### Create a Docker volume
```bash
docker volume create mydata
```
Docker stores volumes in something like:
/var/lib/docker/volumes/

### List all volumes
```bash
docker volume ls
```
![Container Output](screenshots_and_images/vol1.png)

### Inspect the volume
```bash
docker volume inspect mydata
```

### Run a container with the volume
```bash
docker run -dit --name mycontainer -v mydata:/app/data ubuntu
```
This mounts the volume:
mydata → /app/data inside container
![Container Output](screenshots_and_images/vol2.png)

### Check volume content inside container
```bash
docker exec -it mycontainer ls /app/data
```

### Inspect container
```bash
docker inspect mycontainer
```
![Container Output](screenshots_and_images/vol3.png)

## Remove Commands:

### Remove the container
```bash
docker rm -f mycontainer
```

### Remove the volume
```bash
docker volume rm mydata
```

### Remove unused volumes
```bash
docker volume prune
```
